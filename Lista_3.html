Zadanie 1

Ramkowanie

Ze standardowego wejścia pobierany jest ciąg jedynek i zer dowolnej długości. traktowany jako strumień danych. Program rozpoczyna procedurę pakowania. Z uwagi na to, że działanie algorytmu będzie lepiej widoczne jeśli stworzy on jeden pakiet, wejście jest automatycznie dzielone na takie fragmenty (dł. 84 bitów) aby żaden z pakietów nie przekroczył długości 128 bitów.

Do obliczenia CRC potrzebna jest bitowa, a nie tekstowa reprezentacja ciągu, więc program tworzy ją i zapisuje w tablicy typu char dla każdego fragmentu za pomocą następującego kodu, który traktuje każde kolejne 8 bitów strumienia jako liczbę zapisaną w systemie binarnym:

unsigned char header[11];

for (unsigned int i = 0; i < 11; i++)
{
	header[i] = std::stoi(
		part.substr(std::min(i * 8, part.length() - 1), 8), nullptr, 2
	);
}

Na podstawie wcześniej obliczonej tablicy, generowana jest wartość CRC i wstawiana (jako ciąg znaków) na koniec części pakietu. Tak przygotowany ciąg musi teraz przejść przez procedurę rozpychania bitów, ponieważ musimy ustalić jakiś ciąg jedynek i zer, który będzie traktowany jako flaga końca pakietu (w naszym przypadku 01111110) i gdy już taki ciąg ustalimy musimy również upewnić się, że flaga ta nie występuje nigdzie w strumieniu wejściowym (w naszym przypadku wystarczy po każdych pięciu jedynkach dodać zero). Rozpychanie wykonywane jest za pomocą następującej procedury:

std::stringstream ss;
char int_counter = 0;

for (auto it = data.begin(); it != data.end(); it++)
{
	if (*it == '1') int_counter++;
	else int_counter = 0;
	ss << *it;

	if (int_counter == 5)
	{
		int_counter = 0;
		ss << '0';
	}
}

return ss.str();

Tak przygotowany pakiet umieszczany jest na standardowym wyjściu.

Odczytywanie

Tak samo jak w przypadku ramkownia, pobieramy ciąg jedynek i zer ze standardowego wyjścia i dzielimy go na fragmenty, natomiast tym razem, ponieważ nie znamy ich długości, szukamy w ciągu flag wyjścia (01111110) i na ich podstawie dokonujemy podziału.

Kolejnym krokiem jest usunięcie nadmiarowych zer analogiczną do poprzedniej procedurą:

std::stringstream ss;
char int_counter = 0;

for (auto it = data.begin(); it != data.end(); it++)
{
	if (*it == '1') int_counter++;
	else int_counter = 0;
	ss << *it;

	if (int_counter == 5)
	{
		int_counter = 0;
		it++;
	}
}

return ss.str();

Zadanie 2

Protokół CSMA/CD

CSMA/CD czyli Carrier-sense multiple access with collision detection jest protokołem wykorzystywanym gównie w sieciach ethernetowych obsługujących na jednym łączu kilka transmiterów. Jego celem jest wykrywanie i rozwiązywanie kolizji jakie mogą nastąpić gdy dwa urządzenia będą nadawać jednocześnie. W czasie kolizji dane zostaję niekształcone oraz powstaje skok napięcia który można wykryć. Główna procedura protokołu rozpoczyna się gdy dostępny jest pakiet do wysłania i wygląda następująco:

  1. Poczekaj, aż łącze się zwolni
  2. Zacznij transmitować pakiet sprawdzając czy nie wystąpiła kolizja
  3. Jeśli wykryta zostanie kolizja przejdź do następnej procedury
  4. Wyzeruj licznik retransmisji i oczekuj na następny pakiet do wysłania

Natomiast procedura rozwiązywania kolizji to:

  1. Kontynuuj transmisję wysyłając sygnał blokujący, aż osiągnięta zostanie minimalna długość pakietu*
  2. Zwiększ licznik retransmisji o jeden
  3. Jeśli osiągnął on swoją maksymalną wartość (w naszym przypadku 16) zwróć błąd i zakończ działanie.
  4. Oblicz czas oczekiwania*
  5. Wróć do kroku pierwszego głównej procedury

*o tym więcej w następnym podrozdziale

Minimalny rozmiar pakietu oraz czasy oczekiwania

Symulacja

Do symulacji łącza posłuży nam program napisany w języku C++. Przewodem transmisyjnym będzie tablica propagujące wstawione do niej sygnały. Komputerami będą obiekty podłączone na początku, środku i końcu, które z pewnym prawdopodobieństwem będą dostawały nowy pakiet do nadania.

Parametrami wejściowymi symulacji są: długość łącza, prawdopodobieństwo pojawiania się nowych pakietów oraz ich długość (dla uproszczenia wszystkie pakiety są tego samego rozmiaru).

Po zakończeniu działania, czyli po wykonaniu 2000000 iteracji łącza lub gdy nastąpi błąd sieci (pkt 3 procedury rozwiązywania kolizji), na standardowym wyjściu drukowane są następujące statystyki:

  1. Maksymalna wartość licznika retransmisji jaka została osiągnięta (rrr)
  2. Liczba wykonanych iteracji
  3. Liczba, wykrytych przez nadajniki, kolizji (ccc)
